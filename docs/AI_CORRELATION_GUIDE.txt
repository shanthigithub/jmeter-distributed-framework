🤖 OVERVIEW

This framework now includes AI-powered correlation detection using Claude AI.
It intelligently analyzes JMeter test scripts to:

  ✓ Detect dynamic values that need to be extracted
  ✓ Find where those values are reused in subsequent requests  
  ✓ Automatically replace hardcoded values with JMeter variables
  ✓ Generate proper JSONPostProcessor extractors

Similar to Fiddler's AutoCorrelate, but powered by Large Language Models!

═══════════════════════════════════════════════════════════════════════════
  WHAT THE AI DETECTS
═══════════════════════════════════════════════════════════════════════════

The AI looks for common correlation patterns:

✓ SESSION TOKENS
  - access_token, sessionId, session_id
  - JWT tokens (eyJ...)
  - Bearer tokens

✓ SALESFORCE-SPECIFIC
  - aura.token (Aura framework tokens)
  - fwuid (framework unique identifier)
  - context tokens
  - 18-character Salesforce record IDs

✓ SECURITY TOKENS
  - CSRF tokens
  - Nonce values
  - API keys

✓ DYNAMIC VALUES
  - Timestamps
  - Sequence numbers
  - Transaction IDs

═══════════════════════════════════════════════════════════════════════════
  VERIFICATION
═══════════════════════════════════════════════════════════════════════════

After AI enhancement:

1. OPEN IN JMETER
   
   Open the -ai-enhanced.jmx file in JMeter GUI

2. CHECK EXTRACTORS

   Look for JSONPostProcessor elements with "AI: Extract" prefix
   Verify they're attached to the correct requests

3. CHECK VARIABLE USAGE

   Search for ${aura_token}, ${session_id}, etc.
   Verify hardcoded values are replaced

4. RUN TEST

   Run with 2+ threads to verify each thread gets its own values
   Check View Results Tree to see variable values

5. REVIEW REPORT

   Open the -correlation-report.json file
   See detailed analysis of what was found

═══════════════════════════════════════════════════════════════════════════
  TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════

ISSUE: "ANTHROPIC_API_KEY not set"
FIX: Set environment variable with your Claude API key

ISSUE: "No correlations detected"
REASON: Your test may already have proper correlations, or responses
        aren't captured in the JMeter file
FIX: Use this during test generation from Playwright, not on existing
     JMeter files without response data

ISSUE: AI finds wrong correlations
FIX: Review the correlation report and manually adjust the JMeter file.
     AI is smart but not perfect - verify critical correlations

ISSUE: Some values still hardcoded
REASON: AI may have missed edge cases or URL-encoded variations
FIX: Manually add extractors or run the script multiple times

═══════════════════════════════════════════════════════════════════════════
  ADVANCED: PROGRAMMATIC USAGE
═══════════════════════════════════════════════════════════════════════════

Use the AI analyzer directly in your code:

const { AICorrelationAnalyzer } = require('./lib/ai-correlation-analyzer');

async function analyzeTest() {
  const analyzer = new AICorrelationAnalyzer(process.env.ANTHROPIC_API_KEY);
  
  // Add API calls
  analyzer.addAPICall({
    method: 'POST',
    url: 'https://api.example.com/login',
    body: '{"username":"test"}',
    responseBody: { access_token: 'abc123' },
    status: 200
  });
  
  analyzer.addAPICall({
    method: 'GET', 
    url: 'https://api.example.com/data',
    body: null,
    responseBody: { data: [...] },
    status: 200,
    headers: { 'Authorization': 'Bearer abc123' }
  });
  
  // Analyze
  const correlations = await analyzer.analyzeCorrelations();
  
  // Generate report
  await analyzer.generateReport('./output/report.json');
}
